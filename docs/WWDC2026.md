# What changed at WWDC 2026 — and how this repo reflects it

Apple's WWDC 2026 keynote and Platforms State of the Union (June 8, 2026)
reshaped the on-device AI story. This repo is built around the parts that matter
for a perceive→reason pipeline. Sources are linked at the bottom.

## The headline: a model-abstraction layer

Foundation Models gained a `LanguageModel` protocol — a shared interface that
Apple's on-device model, Google Gemini, and Anthropic Claude all implement. You
write the session logic once and swap the backing model by changing one line.
Apple's framing: the protocol is the moat, not the model — Apple wants to be the
on-device orchestration layer regardless of which model actually runs.

**In this repo:** `pipeline/reasoning/language_model.py` (Python) and
`app/.../Reasoning/ReasoningProvider.swift` (Swift) are direct mirrors of that
protocol, with on-device and Claude implementations behind it.

## Other relevant updates

- **Image input (multimodal).** Foundation Models can now take images alongside
  text — but on-device image input requires the new ~20B-parameter sparse
  *AFM 3 Core Advanced* model, which ships on higher-end devices only (roughly
  iPhone 15 Pro and newer). Lower-end devices still get text-only on-device.
- **Free Private Cloud Compute** access for developers with smaller App Store
  footprints, removing the infra cost of server-side inference.
- **Built-in system tools:** a Spotlight-backed RAG tool (on-device retrieval
  with no vector DB), an OCR tool, and a barcode/QR tool — all feeding results
  into the model's context.
- **Python SDK + Linux runtime (new, day-one, treat as beta).** Apple described
  a Python SDK and a Swift-runtime path for Foundation Models on Linux servers,
  aimed at researchers/tooling rather than production serving. **This repo does
  not depend on it** — the day-one package details circulating come from
  secondary/AI-written coverage and are unverified. If/when an official package
  is confirmed, it would slot in as another `LanguageModel` provider with no
  change to the orchestrator. (That is the whole point of the abstraction.)
- **Core ML → "Core AI".** Apple signalled a Core ML successor ("Core AI") for
  third-party model *weights*; Core ML continues in compatibility mode. This
  repo targets the stable Core ML conversion path (`coremltools`), which remains
  the supported way to ship a custom vision model on device.

## What Apple still does *not* give you

- **No custom weights in the on-device Foundation Models runtime.** It is
  Apple's model or a cloud provider; there is no "bring your own fine-tune" for
  on-device FM. Custom weights are the Core ML / Core AI path — which is exactly
  why this repo keeps perception (your model, via Core ML) separate from
  reasoning (Apple's model or Claude, via the abstraction).
- **Regional limits.** Some Apple Intelligence end-user features are restricted
  in the EU and China; the developer Foundation Models API is less affected, but
  verify for your target market.
